kube-console has been deployed.

1. Access the UI:
{{- if .Values.ingress.enabled }}
{{- range .Values.ingress.hosts }}
   https://{{ .host }}
{{- end }}
{{- else }}
   kubectl port-forward svc/{{ include "kube-console.fullname" . }} 8080:{{ .Values.service.port }}
   open http://localhost:8080
{{- end }}

2. Sign in with a Kubernetes bearer token, e.g.:
   kubectl create token <serviceaccount> --duration=1h

The backend holds NO Kubernetes credentials: every request is authorized by
the cluster with the user's own token, and RBAC/audit happen in kube-apiserver.

If the exec terminal disconnects behind an ingress, raise the WebSocket
read/send timeouts on the ingress controller (see values.yaml).
{{- if eq (include "kube-console.caConfigMap" .) "kube-root-ca.crt" }}

The apiserver CA is mounted from the `kube-root-ca.crt` ConfigMap that
Kubernetes auto-publishes in every namespace — no ServiceAccount token, no
RBAC. If the pod stays in ContainerCreating with a "configmap
kube-root-ca.crt not found" event, this cluster does not publish it:

   kubectl -n {{ .Release.Namespace }} get configmap kube-root-ca.crt

Then supply the CA yourself and reinstall with one of:

   --set ca.existingCaConfigMap=<name>   # key: {{ .Values.ca.key }}
   --set ca.existingCaSecret=<name>      # key: {{ .Values.ca.key }}
{{- end }}
